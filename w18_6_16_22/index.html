Typescript (TS)

What is Typescript?

Typescript is a super-set of Javascript. Meaning that it expands on Javascript's functionality. This means that all valid Javascript files are valid Typescript files. The place that Typescript differs is in the features it adds on top of Javascript.

These additional features include:

  • Static Typing (Type checking)

    • If you've ever worked with a statically typed language like C++, C#, Java, etc. then you are familiar with the need to define the types of your variables before you can use them.

  • Code completion
    • By using static typing, you make your code self-aware. Meaning it knows what kind of data it should be receiving or passing around. This allows for much better code completion than your text editor's built-in autocomplete. Instead of getting general Javascript syntax autocompletion you will get a pre-emptive type of autocomplete where it is telling you what variable you need and where.

  • Refactoring
    • Having self-aware code makes refactoring easier as it will often give you editor errors (red underlines) whenever you're making a mistake. This makes mistakes more easily identifiable and also makes them easier to avoid.

  • Shorthand notations
    • Many of the modern shorthands that we use in our workshops weren't always a part of Javascript. Originally, they were implemented in Typescript and were later adopted into the ECMA-script spec.

      These include:
      • Spread/Rest operator (...)
      • Nullish coalescing operator (??)
      • Optional Chaining (object?.method)

Why use Typescript?

Typescript comes in handy to safeguard developers from the quirks of Javascript. Some of these quirks we've encountred in our past workshops. Typescript allows you to define the types your code is expecting. This allows you to make less errors as your code will be self-awate and can tell you when you're trying to use a data type that will make the code break.

When to use Typescript?

Typescript can be used any time you're writing Javascript no matter how small/large the project. However, typescript can be overkill in some cases. If you're building a personal project, you may not need TS. In fact, there are times when TS can make your development flow more complicated than is necessary.

For the most part, TS is used on medium/large projects where there are multiple developers working on one project. This comes in handy as it keeps people from stepping on each other's toes by preventing the misuse of code written by somebody

Typescript Types

Typescript has the primitive types you've come to know from Javascript. However, they've also added additional types so you can more specifically define the data your code needs.
These include:

  • String (string)
  • Number (number)
  • Boolean (boolean)
  • Any (any)


Defining types

To define a variable with a type you simply do the following:

    let num: number = 24

    let name: string = 'Sid'

    let isValid: boolean = false

To define types for a function's argument you do this:

    const myFunction = (arg1: number, arg2: string) => {}

    function myFunction(arg1: number, arg2: string) {}

To define types for a function's return value you do this:

    const myFunction = (arg1: number, arg2: number): number => {
    return arg1 + arg2
}

    function myFunction(name: string): string {
    return `Hello ${name}`
}

Setting up your environment for TS

You should already have node/npm installed so to use typescript all you have to do is run:

    npm install -g typescript

To make sure it installed correctly run:

    tsc -v
